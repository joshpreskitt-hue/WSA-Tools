1

Create the Google Form

Go to forms.google.com and create a new blank form. Name it "WSA Work Order Request" and set the theme color to purple (#271A70). Add the following fields in this exact order — the column mapping in your Sheet depends on this order.

# Field Label Type Options / Notes Required?
1 Your Full Name Short answer — Required
2 Your Email Address Short answer Enable "Response validation → Text → Email address" Required
3 Campus Location Dropdown WSA Main Campus / Satellite Campus Required
4 Category Dropdown HVAC / Plumbing / Electrical / Custodial / IT / Grounds / Safety / Other Required
5 Room / Area Short answer e.g. "Room 204", "Front Office", "Cafeteria" Required
6 Priority Multiple choice Urgent / High / Normal / Low — add brief descriptions to each option Required
7 Description of Issue Paragraph Instruct staff to be as specific as possible Required
8 Photo of Issue (optional) File upload Allow only images. Set max file size to 10 MB. Optional
Tip: On the form settings page, turn on "Collect email addresses" → "Verified" if you want Google to pre-fill the email field for signed-in staff. This reduces typos on the submitter's email.
2

Set Up the Google Sheet

Step 2a

Create a new Google Sheet

Go to sheets.google.com and create a new blank spreadsheet. Name it "WSA Work Order Database". Rename the first tab to "Work Orders" (must match exactly).

Step 2b

Share with your team

Share the Sheet with appropriate permissions:

  • JP — Editor access (manage all work orders)
  • Custodial & IT leads — Commenter or Viewer access
  • General staff — access the Web App Dashboard instead (read-only, no Sheet access needed)
Important: Do not manually enter data in columns A–I. These are populated automatically by the Google Form. Manually entering data in those columns will break the column alignment and may cause email errors.
3

Install the Apps Script

Step 3a

Open the Script Editor

From your Google Sheet, go to Extensions → Apps Script. This opens the script editor in a new tab.

Step 3b

Paste the script

Delete all existing code in the editor. Open the file WSA_WorkOrder_Script.gs provided with this guide, copy the entire contents, and paste it into the editor.

Step 3c

Update your email address

Near the top of the script, find the WO configuration block and update JP_EMAIL with your actual email address:

JP_EMAIL: 'your.actual.email@waynestem.org', // ← update this FORGETTE_EMAIL: 'todd.forgette@waynestem.org', // already set
Step 3d

Save the project

Click the floppy disk icon or press Ctrl+S (Cmd+S on Mac). Name the project "WSA Work Order System".

4

Run the Setup Functions

Important: Run these two functions one time only. Running them again will delete and recreate all triggers, which is harmless but unnecessary.
Step 4a — Run first

Run setupSheet()

In the Apps Script editor, select setupSheet from the function dropdown at the top, then click the ▶ Run button. This formats your sheet: header colors, column widths, dropdown validations, and date formatting. Grant permissions when prompted.

Step 4b — Run second

Run setupTriggers()

Select setupTriggers from the dropdown and click ▶ Run. This installs 5 automated triggers:

  • Form submit → confirmation email + WO# generation
  • Sheet edit → status-change emails
  • Weekly digest → every Monday at 7 AM
  • Monthly report → 1st of each month at 7 AM
  • Overdue check → every day at 8 AM
Verify triggers: After running setupTriggers(), go to Triggers (clock icon in left sidebar). You should see 5 triggers listed. If you see duplicates, run setupTriggers() once more — it clears duplicates automatically.
5

Link the Form to the Sheet

Step 5a

Open your Google Form

Go back to your form at forms.google.com. Click the Responses tab at the top.

Step 5b

Link to the spreadsheet

Click the green Spreadsheet icon (or "Link to Sheets"). Choose "Select existing spreadsheet" and find your "WSA Work Order Database" sheet. Click Select. Google will link the form responses to the "Work Orders" tab.

Step 5c

Test with a practice submission

Submit a test work order through the form. You should see a new row appear in the Sheet with all form data populated in columns A–I. Within seconds, the script should auto-fill the WO# in column J and set Status to "Open" in column L. You should also receive a confirmation email.

If no WO# appears: The script trigger may not have fired. Go to Apps Script → Run onFormSubmit manually and check the Execution Log for errors.
6

Publish the Web App Dashboard

Step 6a

Deploy as a Web App

In Apps Script, click Deploy → New deployment. Select type: Web app. Configure as follows:

  • Description: WSA Work Order Dashboard v1
  • Execute as: Me (your Google account)
  • Who has access: Anyone within [your domain] (or "Anyone" if you want external access)
Step 6b

Copy the Web App URL

After deploying, Google will provide a Web App URL. Copy it — this is the link you'll share with your team. It will look like:
https://script.google.com/macros/s/AKfycb.../exec

Step 6c

Re-deploy after script changes

Any time you update the script code, you must re-deploy: Deploy → Manage deployments → Edit → New version → Deploy. The URL stays the same.

Bookmark it: Share the Web App URL in your staff communication platform (email, Teams, etc.) and add it as a bookmark to school-issued devices. Staff can view all open work orders from any browser without needing Google account access to the Sheet.
7

Generate & Post QR Codes

Staff should be able to submit work orders in under 60 seconds from their phone. QR codes posted at key locations around each campus make this effortless.

Step 7a

Get the form submission link

Open your Google Form, click the Share button (or the eye/preview icon), and copy the link. It will end in /viewform.

Generate QR Codes — Free Tools

Go to qr-code-generator.com or qrcode-monkey.com. Paste your form URL and download a high-resolution PNG. Set the QR color to WSA Purple (#271A70) for branding consistency.

Step 7b

Print and laminate

Print the QR code on a half-sheet or full sheet, add the text "Submit a Work Order — Scan Here" in WSA branding, and laminate for durability. Post at these recommended locations:

  • Faculty break rooms (both campuses)
  • Front office / main entry
  • Custodial supply closet
  • IT equipment room
  • Staff bulletin boards

Email Automation Reference

Trigger: Form Submitted

Confirmation to Submitter

Sent instantly. Includes WO#, full details summary, and note that they'll receive status updates.

Trigger: Form Submitted

New Submission Alert to JP

Sent instantly. Shows all work order details so you can assign and prioritize without opening the Sheet.

Trigger: Status Changed in Sheet

Status Update to Submitter

Fires when you change the Status column (col L) to anything except Completed. Keeps submitters informed automatically.

Trigger: Status Changed to Completed

Closure Email to Submitter

Fires when Status is set to "Completed." Includes resolution notes from col O. Green-themed header to distinguish from other emails.

Trigger: Every Monday at 7 AM

Weekly Digest to JP

Summary stats (total open, urgent, overdue) + sorted table of all non-completed work orders. Overdue items flagged with ⚠.

Trigger: 1st of Month at 7 AM

Monthly Report to JP + Dr. Forgette

Prior month stats: total, completion rate, by campus, by category, average resolution time. Sent to both you and Dr. Forgette automatically.

Resolution Notes matter: Before changing status to "Completed," always fill in column O (Resolution Notes) first. Whatever you type there appears in the closure email the submitter receives. A brief note like "Replaced HVAC filter and recharged unit" goes a long way professionally.

Daily Operations Guide

When a new work order arrives

  • You'll receive a New Submission Alert email with all details
  • Open the Sheet → find the new row (it will be white/amber at the bottom)
  • Set Assigned To (col K) and Est. Completion (col M)
  • Change Status to "In Progress" — the submitter is automatically notified

When a work order is resolved

  • Fill in Resolution Notes (col O) first — this appears in the closure email
  • Change Status to "Completed" — the script auto-stamps col N and sends the closure email
  • Row will turn green automatically

Using the On Hold status

  • Use "On Hold" when waiting on a vendor, part, or budget approval
  • Row turns red — easy to spot in the Sheet and on the dashboard
  • Add context to Admin Notes (col P) explaining why it's on hold
  • The submitter is notified via email that their request is on hold

Managing the Web App Dashboard

  • Data updates on page refresh — no real-time sync needed
  • Share the dashboard URL in your staff communication channels
  • Staff can filter by campus, category, status, priority, and assignee
  • After any script changes, re-deploy to push updates to the dashboard
